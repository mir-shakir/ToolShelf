    Self-Host n8n for Free Forever: A Complete Google Cloud Setup Guide

    By The ToolShelf Team October 30, 2025 15 min read
    n8nself-hostinggoogle clouddockerautomation

    For the past six months, I've been running over 20 automated workflows on n8n, 24/7, without paying a single cent for hosting. It’s not a secret loophole; it's the power of combining n8n's self-hostable nature with the generous Google Cloud free tier.

    While n8n's cloud offering is fantastic, the cost can be a barrier for developers, hobbyists, or startups just getting their feet wet with automation. The problem is that paid plans, while powerful, can feel like overkill when you're just starting. The solution? Taking control of your own instance on infrastructure that costs you nothing. This guide will show you exactly how to achieve automation freedom.

    Why Self-Host n8n?

    Opting to self-host n8n isn't just about saving money; it's about unlocking a new level of control, privacy, and skill development. Here’s why it's a game-changer:

    • Full Data Control & Privacy: Your data and credentials never leave your server. This is crucial for handling sensitive information and ensures compliance with strict privacy standards.
    • No Workflow or Execution Limits: The only limits are the resources of your virtual machine. You can create as many workflows and run them as often as you need, without worrying about hitting a plan's ceiling.
    • Learn Valuable Infrastructure Skills: Setting up and managing your own server, Docker containers, and networking is an invaluable skill for any developer.
    • It's Actually Free: The Google Cloud free tier includes an e2-micro VM instance, which is more than enough to run n8n for a wide range of personal and small business tasks.

    What You'll Achieve Today

    By the end of this guide, you will have a production-ready, self-hosted n8n instance with:

    • 24/7 Uptime: A continuously running n8n instance on a reliable cloud provider.
    • A Free SSL Certificate: Secure HTTPS access to your n8n dashboard using Let's Encrypt.
    • Persistent Data: Your workflows and credentials will be saved to a persistent volume, so they survive restarts and updates.
    • Auto-Restart on Failure: Docker will ensure your n8n container automatically restarts if it ever crashes.
    • The Grand Total Cost: $0 per month, as long as you stay within the generous free tier limits.

    Prerequisites

    Before we dive in, make sure you have the following:

    • A Google Account to sign up for Google Cloud Platform.
    • Basic knowledge of the command line (navigating directories, running commands).
    • A domain name (optional but highly recommended for SSL). You can get one for about $12/year.
    • About 1-2 hours set aside for the initial setup.

    Step-by-Step Setup Guide

    This is where we get our hands dirty. Follow these steps carefully to get your n8n instance up and running.

    Part A: Google Cloud VM Setup

    1. Create a GCP Account: If you don't have one, sign up for the Google Cloud Free Tier. You may need to provide a credit card for verification, but you won't be charged if you stick to the free tier resources.
    2. Create a New Project: In the GCP Console, create a new project named something like `n8n-hosting`.
    3. Navigate to Compute Engine: In the navigation menu, go to `Compute Engine` > `VM instances`.
    4. Create a VM Instance: Click `CREATE INSTANCE`. Use the following configuration:
      • Name: `n8n-server`
      • Region: Choose a region that offers the e2-micro free tier, like `us-west1` (Oregon), `us-central1` (Iowa), or `us-east1` (South Carolina).
      • Machine type: `e2-micro` (This is part of the free tier).
      • Boot disk: Change the OS to `Ubuntu` and the version to `Ubuntu 22.04 LTS`. Keep the default 10 GB standard persistent disk.
      • Firewall: Check `Allow HTTP traffic` and `Allow HTTPS traffic`.
    5. Click `Create`. After a minute, your VM will be ready. Click the `SSH` button to open a terminal session in your browser.

    Part B: Docker Installation

    Once you're connected via SSH, we need to install Docker and Docker Compose. Run the following commands one by one:

    # Update package lists
sudo apt-get update

# Install prerequisites
sudo apt-get install -y ca-certificates curl gnupg

# Add Docker’s official GPG key
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Set up the repository
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker Engine
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Add your user to the docker group to run docker commands without sudo
sudo usermod -aG docker $USER

# Apply the group changes
newgrp docker

    Verify the installation by running docker --version. For more details on using Docker with AI and GenAI workloads, check out our developer's guide to local LLMs with Docker.

    Part C: n8n Configuration with Docker Compose

    We will use `docker-compose` to define and run our n8n service. It simplifies management, especially for data persistence and environment variables.

    1. Create a directory for your n8n configuration: 
      mkdir n8n-data
cd n8n-data
    2. Create a `docker-compose.yml` file: 
      nano docker-compose.yml
    3. Paste the following configuration into the file. Replace `your.domain.com` with your actual domain name.
    version: '3.7'

services:
  n8n:
    image: n8nio/n8n
    restart: always
    ports:
      - "127.0.0.1:5678:5678"
    environment:
      - N8N_HOST=your.domain.com
      - N8N_PORT=5678
      - N8N_PROTOCOL=https
      - NODE_ENV=production
      - WEBHOOK_URL=https://your.domain.com/
      - GENERIC_TIMEZONE=America/New_York # Change to your timezone
    volumes:
      - ./n8n-local-data:/home/node/.n8n

  caddy:
    image: caddy:2
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
      - caddy_config:/config

volumes:
  caddy_data:
  caddy_config:

    This configuration defines two services: `n8n` itself and `caddy`, a simple web server that will act as a reverse proxy and automatically handle SSL certificates for us.

    Part D: SSL/Domain Setup with Caddy

    1. In your domain registrar's DNS settings, create an `A` record pointing your domain (e.g., `n8n.yourdomain.com`) to the external IP address of your Google Cloud VM.
    2. Back in your SSH session (inside the `n8n-data` directory), create a file named `Caddyfile`: 
      nano Caddyfile
    3. Paste the following configuration, again replacing `your.domain.com` with your domain:
    your.domain.com {
    reverse_proxy n8n:5678
}

    This tells Caddy to listen for traffic on your domain and forward it to the n8n container, handling all the HTTPS and SSL certificate logic automatically.

    Part E: Launch and Secure

    You're now ready to launch!

    1. From the `n8n-data` directory, run: 
      docker compose up -d
      This command will download the images and start your n8n and Caddy containers in the background.
    2. Initial Setup: Navigate to `https://your.domain.com` in your browser. You should see the n8n setup screen. Create your owner account immediately.
    3. Security Hardening: It's crucial to set up proper authentication to protect your instance. In n8n, go to `Settings > Users` to manage access. A robust security strategy is non-negotiable; for a deeper dive, review our guide on hash function security in 2025.

    Monitoring & Maintenance

    Your n8n instance is running, but the job isn't over. Here are some quick tips for keeping it healthy:

    • Check Status: Run docker ps to see if your containers are running.
    • View Logs: Use docker compose logs -f n8n to see the logs for the n8n service in real-time, which is helpful for debugging.
    • Updating n8n: To update to the latest version, run the following commands in your `n8n-data` directory: 
      docker compose pull
docker compose up -d
    • Backup Your Workflows: Your data is in the `./n8n-local-data` directory. Regularly back up this folder to a safe location.
